```
│              │  │  ↑                                     │  │
│              │  │  |  ～～～～～～～～～～～～            │  │
│              │  │  └──────────────────────── 時刻        │  │

│              │  │ 🚶 人感センサー                        │  │
│              │  │  検知: ██░░░░██████░░░██░░░░░░░░░     │  │

│              │  └────────────────────────────────────────┘  │
└──────────────┴──────────────────────────────────────────────┘
```

**仕様:**
- デバイス選択: `GET /api/devices` で取得したデバイス一覧を表示。未選択時は全デバイス
- 期間選択: 直近1時間 / 直近24時間 / 直近7日（デフォルト: 直近24時間）
- グラフデータ: `GET /api/iot-data?device_id=&from=&to=&per_page=1000` で取得
- データがない期間はグラフを空白にする

**デバイス0件時のエンプティステート:**

```
┌──────────────┬──────────────────────────────────────────────┐
│  サイドバー  │  ダッシュボード                              │
│              │                                              │
│              │  デバイス選択: [---- ▼]  期間: [直近24h ▼] │
│              │                                              │
│              │  ┌────────────────────────────────────────┐  │
│              │  │                                        │  │
│              │  │   📱 デバイスが登録されていません      │  │
│              │  │                                        │  │
│              │  │   デバイス管理画面からデバイスを       │  │
│              │  │   追加してください                     │  │
│              │  │                                        │  │
│              │  │          [デバイス管理へ →]            │  │
│              │  │                                        │  │
│              │  └────────────────────────────────────────┘  │
└──────────────┴──────────────────────────────────────────────┘
```

- `GET /api/devices` の結果が0件の場合、グラフエリアをエンプティステートに差し替える
- 「デバイス管理へ →」ボタンは `/devices` へ遷移する

**期間パラメータ計算（JST基準の現在時刻から算出）:**

| 選択肢 | from | to |
|---|---|---|
| 直近1時間 | `now() - 1h` | `now()` |
| 直近24時間 | `now() - 24h` | `now()` |
| 直近7日 | `now() - 7d` | `now()` |

**ページネーション処理（直近7日の場合）:**

直近7日 × 1分間隔 × 複数デバイスでは最大10,080件となり `per_page=1000` では最大10ページが必要になる。フロントエンドは `page` を1から順にインクリメントしながら全ページを順次取得し、結合したデータでグラフを描画する。`next_page` が存在しない（またはレスポンス件数が `per_page` 未満）になった時点で取得完了とする。

# 6. S-03: デバイス管理

```
┌──────────────┬──────────────────────────────────────────────┐
│  サイドバー  │  デバイス管理                                │
│              │                                    [+ 追加]  │
│              │  ┌──────────────────────────────────────┐   │
│              │  │ device_id  │ 表示名   │ 登録日時     │   │
│              │  │──────────────────────────────────────│   │
│              │  │ room01     │ リビング │ 2026-01-01   │ ✏️ 🗑️│
│              │  │ kitchen    │ キッチン │ 2026-01-05   │ ✏️ 🗑️│
│              │  └──────────────────────────────────────┘   │
└──────────────┴──────────────────────────────────────────────┘
```

**[+ 追加] モーダル:**
```
┌─────────────────────────────────┐
│  デバイスを追加                  │
│                                 │
│  デバイスID *                   │
│  ┌─────────────────────────┐   │
│  │ room01                  │   │
│  └─────────────────────────┘   │
│                                 │
│  表示名                         │
│  ┌─────────────────────────┐   │
│  │ リビング                │   │
│  └─────────────────────────┘   │
│                                 │
│        [キャンセル]  [追加する]  │
└─────────────────────────────────┘
```

**✏️（編集）モーダル:**
```
┌─────────────────────────────────┐
│  表示名を変更                    │
│                                 │
│  表示名                         │
│  ┌─────────────────────────┐   │
│  │ リビング                │   │
│  └─────────────────────────┘   │
│                                 │
│        [キャンセル]  [保存する]  │
└─────────────────────────────────┘
```

**🗑️（削除）確認ダイアログ:**
```
┌─────────────────────────────────┐
│  デバイスを削除しますか？        │
│                                 │
│  「room01」を削除します。        │
│  関連するIoTデータ・異常ログも   │
│  すべて削除されます。            │
│                                 │
│        [キャンセル]  [削除する]  │
└─────────────────────────────────┘
```

**仕様:**
- 追加: `POST /api/devices`。409の場合「このデバイスIDはすでに登録されています」を表示
- 編集: `PATCH /api/devices/{id}`（表示名のみ変更可。device_idは変更不可）
- 削除: `DELETE /api/devices/{id}`

> **`id` と `device_id` の使い分け:** PATCH/DELETE のパスパラメータ `{id}` は `devices.id`（UUID）を使用する。`GET /api/devices` のレスポンスに含まれる `id` フィールドをフロントエンドで保持し、編集・削除時にパスへ使用する。ワイヤーフレーム上に表示している `device_id`（文字列）とは別物であり、パスには使用しない。

```
┌──────────────┬──────────────────────────────────────────────┐
│  サイドバー  │  レポート詳細: 2026-01-14          [← 戻る] │
│              │                                              │
│              │  ┌────────────┐ ┌────────────┐             │
│              │  │ 総消費電力 │ │ 平均温度   │             │
│              │  │  3.52 kWh  │ │  22.1 ℃   │             │
│              │  └────────────┘ └────────────┘             │
│              │  ┌────────────┐ ┌────────────┐             │
│              │  │ 平均湿度   │ │ 在室時間   │             │
│              │  │  58.3 %    │ │  420 分    │             │
│              │  └────────────┘ └────────────┘             │
│              │                                              │
│              │  異常検知サマリー                            │
│              │  ┌──────────────────────────────────────┐   │
│              │  │ デバイス │ 検知項目 │ 件数 │ 最大値   │  │
│              │  │──────────────────────────────────────│   │
│              │  │ room01   │ 温度     │ 2件  │ 38.2℃   │  │
│              │  └──────────────────────────────────────┘   │
│              │                                              │
│              │  [📥 PDFダウンロード]  ※本日 1/3 回使用済み │
└──────────────┴──────────────────────────────────────────────┘
```

**仕様:**
- `GET /api/reports/{id}` でデータ取得
- 値が NULL の場合は `--` と表示
- PDFダウンロード: `POST /api/reports/{id}/download`
  - `download_count_today` が 3 の場合、ボタンを非活性にして「本日のダウンロード上限（3回）に達しました」を表示
  - HTTP 200 を受け取った時点でダウンロード成功とみなし、`GET /api/reports/{id}` を再コールしてカウント表示を更新する
  - 失敗時（500 等）はカウントを更新しない。「ダウンロードに失敗しました。再度お試しください」をトースト表示する
  - 429エラー時は「本日のダウンロード上限に達しました」をトースト表示
- [← 戻る] ボタン: `/reports` へ固定遷移する（`history.back()` は使用しない。直接URL アクセス時も一貫した遷移先を保証するため）

- power_w の編集モーダルでは「下限」項目を非表示にする
- 有効/無効トグル: `PATCH /api/anomaly-thresholds/{id}` で `enabled` のみ更新
