# ドキュメントレビュー

引数: ドキュメントが存在するディレクトリパス(例: `/review-basic-design-docs basic_design/sequence`)

## 実行方法

```bash
claude
> /review-basic-design-docs basic_design/sequence
```

## 手順

### ステップ1: ドキュメントの存在確認

指定されたフォルダにmdドキュメントが存在するか確認します。

### ステップ2: basic-design-doc-reviewerサブエージェント起動

basic-design-doc-reviewerサブエージェントを起動してディレクトリパス配下にあるすべてのドキュメントについて、レビューを実行します:

Task toolを使用してbasic-design-doc-reviewerサブエージェントを起動してください:
- subagent_type: "basic-design-doc-reviewer"
- description: "Basic design document detailed review"
- prompt: "[ドキュメントパス]を詳細にレビューしてください。\n\n以下の観点で評価してください:\n1. 完全性: 必要な項目が全て含まれているか\n2. 具体性: 曖昧な表現がないか\n3. 一貫性: 他のドキュメントと整合しているか\n4. 測定可能性: 成功指標が測定可能か(PRDの場合)\n\nレビューレポートを作成してください。"

### ステップ3: レビュー結果の要約

サブエージェントが作成したレビューレポートの要点を抽出し、ユーザーに報告します。

## 出力形式

```markdown
# ドキュメントレビュー結果

## ドキュメント: [ファイル名]

### 主な改善点

1. [改善点1] (優先度: 高/中/低)
2. [改善点2] (優先度: 高/中/低)
3. [改善点3] (優先度: 高/中/低)

### 総合評価

[1-5]/5

### 次のアクション

- [推奨対応1]
- [推奨対応2]

詳細なレポートはサブエージェントの出力を参照してください。
```

### 出力場所
レポートの内容は、home-smart-factory/reviewに保存されます。ファイル名は`[ドキュメント名]_review.md`としてください。

## 注意事項

- レビューは詳細な分析のため、数分かかる場合があります
- サブエージェントは独立したコンテキストで動作するため、メインエージェントのコンテキストは消費しません
