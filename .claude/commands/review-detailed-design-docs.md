# ドキュメントレビュー

引数: ドキュメントが存在するディレクトリパス(例: `/review-detailed-design-docs docs/detailed-design/`)

## 実行方法

```bash
claude
> /review-detailed-design-docs docs/detailed-design/
```

## 手順

### ステップ1: ドキュメントの存在確認

指定されたフォルダにmdドキュメントが存在するか確認します。

### ステップ2: detailed-design-doc-reviewerサブエージェント起動

detailed-design-doc-reviewerサブエージェントを起動してディレクトリパス配下にあるすべてのドキュメントについて、レビューを実行します:

Task toolを使用してdetailed-design-doc-reviewerサブエージェントを起動してください:
- subagent_type: "detailed-design-doc-reviewer"
- description: "Detailed design document review"
- prompt: "[ドキュメントパス]を詳細にレビューしてください。\n\n以下の観点で評価してください:\n1. 実装可能性: 擬似コード・アルゴリズムレベルで具体的に記述されているか\n2. 具体性: 曖昧な表現がなく、実装者が迷わない粒度か\n3. 基本設計との整合性: docs/basic-design/ および docs/spec-requirements/ の内容と矛盾がないか\n4. エッジケース網羅性: 異常系・境界値・エラーケースが考慮されているか\n\nレビューレポートを作成してください。"

### ステップ3: レビュー結果の要約

サブエージェントが作成したレビューレポートの要点を抽出し、ユーザーに報告します。

## 出力形式

```markdown
# ドキュメントレビュー結果

## ドキュメント: [ファイル名]

### 主な改善点

1. [改善点1] (優先度: 高/中/低)
2. [改善点2] (優先度: 高/中/低)
3. [改善点3] (優先度: 高/中/低)

### 総合評価

[1-5]/5

### 次のアクション

- [推奨対応1]
- [推奨対応2]

詳細なレポートはサブエージェントの出力を参照してください。
```

### 出力場所
レポートの内容は、プロジェクトルートの`review/`ディレクトリに保存されます。ファイル名は`[ドキュメント名]_review.md`としてください。

## 注意事項

- レビューは詳細な分析のため、数分かかる場合があります
- サブエージェントは独立したコンテキストで動作するため、メインエージェントのコンテキストは消費しません
